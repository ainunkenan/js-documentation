Javascript Documentation

Introduction

JavaScript is a powerful and versatile programming language that plays a pivotal role in the world of web development.
In 1995, JavaScript was created by Brendan Eich in just ten days while working at Netscape Communications. Initially known as "Mocha" it was later renamed "LiveScript" before finally becoming JavaScript.
Its purpose was to add client-side interactivity to web pages, and it was first introduced in the Netscape Navigator browser.

Recognizing the need for a standardized version of the language, Netscape submitted JavaScript to the Ecma International organization. This led to the first ECMAScript standard, published in 1997.
ECMAScript is the formal specification for JavaScript. JavaScript's popularity grew rapidly, but it faced compatibility issues due to competing browser vendors implementing the language differently.
This period, known as the "browser wars," marked significant advancements in JavaScript capabilities and performance.

JavaScript became a crucial part of the Web 2.0 movement. Technologies like Asynchronous JavaScript and XML (AJAX) allowed for seamless data exchange between the client and server, ushering in an era of dynamic and interactive web applications.

JavaScript became an integral part of the HTML5 standard, allowing for new capabilities like the HTML5 Canvas, Web Workers, and the Geolocation API. These additions revolutionized web development, making it possible to create complex applications entirely in the browser.

JavaScript saw the emergence of powerful front-end and back-end frameworks, such as Angular, React, and Vue.js for client-side development, and Express.js for server-side development with Node.js. These frameworks streamlined the development process and offered robust solutions for building modern web applications.

ECMAScript 6 (ES6) was a significant update to JavaScript, introducing new syntax and features that enhanced the language's expressiveness and maintainability. Subsequent ECMAScript releases have continued to add features and improvements.

Today, JavaScript is one of the most widely used programming languages globally, powering not only web applications but also mobile apps (through frameworks like React Native) and even desktop applications. Its extensive ecosystem, active community, and continuous evolution make it a fundamental tool for developers in the ever-evolving world of technology.

JavaScript vs Java,
Unraveling the Differences

JavaScript and Java, despite their similar names, are two distinct programming languages with different purposes, use cases, and characteristics.

  1. Purpose and Use Cases:
    • JavaScript: JavaScript is primarily a client-side scripting language used for enhancing web pages. It allows developers to create interactive and dynamic content in web browsers. JavaScript is essential for tasks such as form validation, animations, and modifying the Document Object Model (DOM) on web pages.
    • Java: Java is a versatile, general-purpose programming language that can be used for a wide range of applications, including web development, desktop software, mobile apps, and backend systems. It is often used for building Android mobile applications, enterprise-level applications, and large-scale, server-side applications.

  2. Language Type:
    • JavaScript: JavaScript is a dynamically typed, interpreted language. It does not require compilation and is executed directly in the browser.
    • Java: Java is a statically typed language, which means that type checking is performed at compile time. Java code is compiled into bytecode, which runs on the Java Virtual Machine (JVM).

  3. Syntax and Semantics:
    • JavaScript: JavaScript's syntax is inspired by C and C++, but it also has functional programming features. It has a flexible and forgiving syntax, making it accessible for beginners. It supports event-driven, asynchronous programming, which is crucial for web development.
    • Java: Java's syntax is more rigid and verbose compared to JavaScript. It follows a more traditional object-oriented programming (OOP) paradigm and enforces strong typing and explicit declarations.

  4. Ecosystem and Libraries:
    • JavaScript: JavaScript has a vast ecosystem of libraries and frameworks, including jQuery, React, Angular, and Vue.js for front-end development, and Node.js for server-side development. This rich ecosystem simplifies and accelerates web and application development.
    • Java: Java also boasts a robust ecosystem with numerous libraries and frameworks. Popular Java frameworks include Spring for web applications and Hibernate for database access. It is widely used in enterprise environments.

  5. Platform and Runtime:
    • JavaScript: JavaScript runs within web browsers, making it a client-side language. However, with Node.js, it can also be used on the server-side.
    • Java: Java applications are typically compiled into bytecode that runs on the Java Virtual Machine (JVM). This enables Java to be platform-independent, as long as there is a compatible JVM for the target platform.

  6. Learning Curve:
    • JavaScript: JavaScript is often considered more accessible to beginners due to its forgiving nature (e.g dynamically typed, etc) and extensive online resources for learning.
    • Java: Java's learning curve can be steeper, particularly for those new to programming. Its static typing and OOP principles may require more upfront knowledge and effort to master.

While JavaScript and Java share a name and some syntactical similarities, they serve different purposes and have distinct characteristics. JavaScript is the go-to language for web development and enhancing user interfaces, while Java is a versatile language used in a wide array of applications, from mobile apps to enterprise-level software.

Variables

Variables are a fundamental concept in JavaScript, as in most programming languages. They act as containers for storing and managing data within your scripts. JavaScript variables are versatile, allowing you to work with various types of data, change their values, and use them in calculations and logical operations.

Declaring Variables:

In JavaScript, you can declare variables using three keywords: var, let, and const.

  • var: Historically, var was the primary way to declare variables in JavaScript. Variables declared with var, are either function-scoped or global-scoped. They are visible through blocks.
  • let: Introduced in ECMAScript 6 (ES6), let provides block-level scope, making it a better choice for declaring variables within blocks, such as loops and conditional statements. Variables declared with let can be reassigned.
  • const: Also introduced in ES6, const is used to declare constants, which are variables with values that cannot be changed after assignment. Like let, const also has block-level scope.

Variable Naming Rules:

In JavaScript, variable names are subject to specific naming rules and conventions. Adhering to these rules ensures your code is readable, maintainable, and less error-prone.

  • Variable names are case-sensitive, meaning myName and myname are considered distinct variables.
  • The first character must be a letter, an underscore, or a dollar sign.
  • Variable names can consist of letters, digits, underscores (_), and dollar signs ($).
  • JavaScript has reserved words, also known as keywords, that cannot be used as variable names. These reserved words are part of the language's syntax and have predefined meanings. Examples include if, else, while, function, and others. You cannot use these as variable names.
  • Variable names should be descriptive and convey the purpose of the variable. Meaningful variable names make your code more understandable and maintainable.

    For example, instead of using x or a as a variable name, use something like totalPrice or userAge to indicate what the variable represents.
    Avoid using single letters, such as i, j, or x, as variable names for non-trivial variables. Single-letter variable names can be confusing and should be reserved for loop counters in short loops.

  • JavaScript developers often use two common conventions for naming variables: camelCase and snake_case.
    • CamelCase : the first word is in lowercase, and subsequent words begin with uppercase letters. For example: myVariableName or userName.
    • Snake_case : words are separated by underscores. For example: my_variable_name or user_name.
  • Constants are often named in ALL_CAPS with underscores, such as MAX_VALUE or PI. This naming convention indicates that the value should not be changed after assignment.
  • It's a good practice to include a data type hint in your variable names. For instance, if a variable stores a date, you can include "date" in the name, such as startDate or endDate.
  • While JavaScript allows some special characters in variable names, it's best to avoid them to maintain code clarity and avoid potential issues. Stick to letters, digits, underscores, and dollar signs.

Assigning Values:

You can assign values to variables during declaration or later in your code.
For example: 

          
let user_age = 28; // Declaration and assignment
let userName; // Declaration without assignment
userName = "Kenan"; // Assignment after declaration

Variables Scope

In JavaScript, the scope of a variable refers to the region or context in which that variable is accessible and can be used. The scope of a variable is determined by how and where it is declared.
There are two main types of variable scope in JavaScript: global scope and local (or function) scope.

  1. Global Scope
    • Variables declared outside of any function or block have global scope. They are accessible from anywhere in the JavaScript code, both inside and outside functions.
    • Be cautious with global variables, as they can lead to naming conflicts and make it harder to track variable changes in large applications. They should be used sparingly and for values that genuinely need to be accessible globally.

    Example of a global variable:

                
var globalVar = 10;
function myFunc () {
  // globalVar is accessible here
  console.log(globalVar);
}

myFunc(); // Outputs 10
console.log(globalVar); // Outputs 10
  2. Local (Function) Scope
    • Variables declared within a function have local scope, meaning they are only accessible within that function.
    • Local scope also applies to variables declared within block statements, such as if statements or loops, when using let or const.

    Example of a local variable:

                
function myFunc () {
  var localVar = 10; // localVar is locally scoped
  console.log(localVar);
}
            
myFunc(); // Outputs 10
console.log(localVar); // ReferenceError: localVar is not defined
  3. Block Scope
    • In modern JavaScript, variables declared with let and const have block-level scope. This means they are confined to the block in which they are declared, whether it's a function, if statement, loop, or any other block.
    • Block-scoped variables are not hoisted, meaning they are only accessible after their declaration within the block.

    Example of block-scoped variables:

                
function myFunc () {
  if (true) {
    let blockVar = 30; // blockVar is block-scoped
    console.log(blockVar); // Outputs 30
  }

  console.log(blockVar); // ReferenceError: blockVar is not defined
}

Let Var Go

The var keyword was the primary way to declare variables in JavaScript for a long time.
However, with the introduction of ECMAScript 6 (ES6) and the availability of let and const, there are strong reasons to let go of the var keyword and embrace the newer variable declaration options.


The Problem With Var

  1. No Block Scope

    2. Variables declared with var, are either function-scoped or global-scoped. They are visible through blocks.
    For Example : 

                
if (true) {
  var fruit = "Grape";
}

console.log(fruit); // Outputs Grape, the variable lives after if

    As var ignores code blocks, we've got a global variable fruit. But if we used let fruit instead of var fruit, then the variable would only be visible inside if. 

                
if (true) {
  let fruit = "Grape";
}

console.log(fruit); // ReferenceError: fruit is not defined

    Another example :

                
for (var i = 0 ; i < 10 ; i++) {
  var one = 1;
  //....
}

console.log(i); // 10, "i" is visible after loop, it's a global variable
console.log(one); // 1, "one" is visible after loop, it's a global variable

    As we can see, var pierces through if, for or other code blocks. That's because a long time ago in JavaScript, blocks had no Lexical Environments, and var is a remnant of that time.

  2. Tolerates Re-declaration

    3. We can redeclare a variable any number of times with var.
    For Example : 

                
var name = "Kenan";
var name = "Kenny"; // the "name" variable is being redeclared
// ...it doesn't trigger an error

console.log(name); // Kenny

    But if we declare the same variable with let twice, an error happened : 

                
let name = "Kenan";
let name = "Kenny"; // SyntaxError: 'name' has already been declared
  3. Declared below their use

    4. var declarations are processed when the function starts (or script starts for globals).
    In other words, var variables are defined from the beginning of the function, no matter where the definition is (assuming that the definition is not in the nested function).
    Example :

                
function sayHi () {
  name = "Kenan";
  console.log ("Hi" + " " + name);
  var name;
}

sayHi(); // Output Hi Kenan

    While, if we do the same with let : 

                
function sayHi () {
  name = "Kenan";
  console.log ("Hi" + " " + name);
  let name;
}
                
sayHi(); // ReferenceError: Cannot access 'name' before initialization

    People also call such behavior “hoisting” (raising), because all var are “hoisted” (raised) to the top of the function.
    But, it is important to notice, while the declarations are “hoisted”, the assignments are not.
    For Example :

                
function sayHi () {
  console.log ("Hi" + " " + name);
  var name = "Kenan";
}
                
sayHi(); // Output Hi undefined

    The line var name = "Kenan"; has two action in it

    1. Variable Declaration : var name
    2. Variable Assignment : = "Kenan"

    Another Example :

                
function sayHi () {
  console.log ("Hi" + " " + name); // Hi undefined
  var name; // declaration is being “hoisted”..
  name = "Kenan"; // ..assignment works when the execution reaches it
}

sayHi(); // Output Hi undefined

    Because all var declarations are processed at the function start (“hoisted”), we can reference them at any place.
    But variables are undefined until the assignments.

    In both examples above, the code runs without an error, because the variable name exists.
    But its value is not yet assigned, so it shows undefined.

    In summary, the var keyword in JavaScript has been largely replaced by let and const due to their block-level scoping, the prevention of hoisting-related issues, and improved code predictability.

    By embracing let and const, we can write cleaner, more maintainable, and less error-prone JavaScript code.
    It's advisable to transition away from var in modern JavaScript development.

Data Types

Data types are fundamental building blocks in any programming language, including JavaScript. They define the kind of data that can be stored and manipulated in your JavaScript code.
Understanding data types is essential for writing effective and error-free JavaScript programs.
In JavaScript, data types are categorized into two main groups: primitive data types and reference data types.

  1. Primitive Data Types
    • Number: Represents both integers and floating-point numbers.
      Examples: 42, 3.14.
    • String: Represents sequences of characters, enclosed in single (' '), double (" "), or backticks (` `) quotes.
      Examples: 'Hello, world!', "JavaScript".
    • Boolean: Represents true or false values.
      Examples: true, false.
    • Undefined: Represents a variable that has been declared but has not been assigned a value.
    • Null: Represents the absence of a value. It is often used to explicitly indicate that a variable is empty or has no value.
    • Symbol (ES6 and later): Represents unique and immutable values. Symbols are often used as object property keys.

  2. Reference Data Types

    3. Reference data types are more complex and can hold multiple values and methods. They are mutable, which means their contents can be changed. JavaScript has three primary reference data types.

    • Object: Objects are complex data structures that can store key-value pairs. Objects can be used to represent a wide range of data, from simple data structures to more complex entities like arrays, functions, and custom data types.
    • Array: Arrays are a specific type of object used to store ordered collections of values. Elements in an array can be of any data type and are accessed by their index.
    • Function: Functions are also objects in JavaScript. They can be assigned to variables, passed as arguments to other functions, and returned as values. Functions are used to define reusable blocks of code.
